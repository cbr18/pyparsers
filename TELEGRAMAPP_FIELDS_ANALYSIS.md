# Анализ полей, выводимых в telegramapp

## Поля, выводимые в CarCard (карточка в списке)

### Всегда выводимые:
- `image` - главное изображение
- `title` / `car_name` - название
- `final_price` / `rub_price` - цена (если > 0)
- `year` - год выпуска
- `brand_name` - бренд

### Условно выводимые (если есть значение):
- `mileage` - пробег (если > 0)
- `city` - город (если есть)
- `series_name` - серия (если есть)
- `car_name` - модель (если есть и отличается от title)

---

## Поля, выводимые в CarDetails (детальная страница)

### Основная информация:
- `image_gallery` - галерея изображений (через ImageCarousel)
- `title` / `car_name` - название
- `final_price` / `rub_price` - цена
- `year` - год выпуска
- `mileage` - пробег (если > 0)
- `city` - город (если есть)
- `source` - источник (dongchedi/che168)

### Расчет стоимости:
- `rub_price` - цена в рублях
- `recycling_fee` - утильсбор
- `customs_duty` - таможенная пошлина
- `customs_fee` - таможенный сбор
- Комиссия (фиксированная: 80000)
- Таможенный брокер (фиксированная: 80000)
- Итого ориентировочно

### Марка и модель:
- `brand_name` - бренд
- `series_name` - серия
- `car_name` - модель

### Технические характеристики:
- `engine_volume` - объем двигателя (л)
- `power` - мощность (л.с.)
- `torque` - крутящий момент (Н⋅м)
- `transmission_type` / `transmission` - КПП
- `drive_type` - привод
- `fuel_type` - тип топлива
- `fuel_consumption` - расход топлива (л/100км)
- `acceleration` - разгон 0-100 (сек)
- `max_speed` - максимальная скорость (км/ч)

### Размеры и вес (если есть хотя бы одно):
- `length` - длина (мм)
- `width` - ширина (мм)
- `height` - высота (мм)
- `wheelbase` - колесная база (мм)
- `curb_weight` - снаряженная масса (кг)

### Кузов и экстерьер (если есть хотя бы одно):
- `body_type` - тип кузова
- `exterior_color` / `color` - цвет
- `door_count` - количество дверей
- `seat_count` - количество мест

### Дополнительная информация (если есть хотя бы одно):
- `condition` - состояние
- `owner_count` - количество владельцев (если > 0)
- `emission_standard` - экологический стандарт
- `certification` - сертификация

### Прочее:
- `description` - описание (если есть)
- `link` - ссылка на источник

## Поля, доступные в модели Car, но НЕ выводимые в telegramapp

### Базовые поля:
- `uuid` - используется для навигации, но не отображается
- `car_id` - внутренний ID

## Итого

### Выводится в telegramapp:
- **CarCard**: ~9 полей (из них 4 условных)
- **CarDetails**: ~35 полей (многие условные)

### НЕ выводится в telegramapp:
- **~80+ полей** из модели Car

### Рекомендации:
1. Большинство технических деталей (двигатель, подвеска, тормоза) не отображаются
2. Характеристики безопасности и комфорта не показываются
3. История автомобиля (ДТП, обслуживание) не отображается
4. Мультимедиа и опции не показываются
5. Метаданные (просмотры, избранное) не используются

Если нужно добавить отображение каких-то полей, их можно добавить в соответствующие компоненты (`CarCard.js` или `CarDetails.js`).
