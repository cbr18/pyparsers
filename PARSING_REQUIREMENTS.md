# Требования к новой архитектуре парсинга

## 🎯 Основная концепция

**Двухэтапный процесс:**
1. **Этап 1 (Список)**: Получить только список ID машин - **НЕ сохранять в БД**
2. **Этап 2 (Детальный)**: Сразу парсить детальную информацию для каждого ID - **сохранять только результат детального парсинга**

---

## 📋 Требования

### 1. Инкрементальный парсинг (приоритет новых машин)

**Требование:** Новые машины должны парситься первыми, не нужно парсить все ID сразу.

**Реализация:**
- Парсить страницы последовательно (page 1, 2, 3...)
- Для каждой страницы: получить список ID → сразу детальный парсинг
- Останавливаться при первом совпадении с существующими ID (для инкрементального режима)
- В полном режиме: парсить все страницы, но последовательно
- **При совпадении с существующим ID: ПРОПУСКАТЬ** (не обновлять)
- **Оптимизировать чтобы старые фрагменты не парсились заново**

**Вопросы:**

### 2. Обработка ошибок детального парсинга

**Требование:** Пропускать машину, если детальный парсинг не удался.

**Реализация:**
- Если `fetch_car_detail()` / `parse_car_details()` вернул `None` или ошибку → пропустить машину
- Логировать ошибки для мониторинга
- Не сохранять в БД машины без детальной информации

**Вопросы:**

### 3. Валидация и удаление неактуальных машин

**Требование:** Проверять, есть ли машины у источников, и помечать те, которые купили или убрали.

**Реализация:**
- **Отдельный воркер** для валидации (не в процессе парсинга)
- Периодически проверять машины из БД: делать запрос к источнику по car_id/sku_id
- Если машина не найдена или помечена как "продана" → **пометить `is_available = false`** (не удалять)

**Вопросы:**

### 4. Фильтрация по году

**Требование:** Фильтровать машины по году (min_year=2017).

**Реализация:**
- **Фильтровать ДО детального парсинга** (экономия времени)
- Если год есть в списке и >= 2017 → парсить детально
- Если год < 2017 → пропустить, не парсить детально

## 🔄 Предлагаемый workflow

### FULL режим:
```
1. Загрузить все существующие ID из БД в Set (оптимизация)
2. Для каждой страницы (page 1, 2, 3...):
   a. fetch_cars_by_page(page) → список машин с ID
   b. Фильтровать по году ДО детального парсинга (если год < 2017 → пропустить)
   c. Проверить существование ID в Set (если есть → пропустить)
   d. Для новых машин (параллельно, лимит 5 для каждого источника):
      - fetch_car_detail(sku_id) / parse_car_details(car_id)
      - Если успешно → добавить в буфер
      - Если ошибка → пропустить, залогировать
   e. Отправить буфер в БД (батчами по 50-100)
3. Завершение
```

### INCREMENTAL режим:
```
1. Загрузить existing_ids в Set
2. Для каждой страницы (page 1, 2, 3...):
   a. fetch_cars_by_page(page) → список машин с ID
   b. Фильтровать по году ДО детального парсинга (если год < 2017 → пропустить)
   c. Проверить, есть ли ID в Set
   d. Если найдено совпадение → остановиться
   e. Для новых машин (параллельно, лимит 5 для каждого источника):
      - fetch_car_detail(sku_id) / parse_car_details(car_id)
      - Если успешно → добавить в буфер
      - Если ошибка → пропустить, залогировать
   f. Отправить буфер в БД (батчами по 50-100)
3. Завершение
```

### Валидация (отдельный воркер в datahub/):
```
1. Получить список машин из БД (source = 'dongchedi' или 'che168', is_available = true)
2. Обработать батчами (по 50-100 машин):
   a. Для каждой машины в батче (параллельно, лимит 5 для каждого источника):
      - Проверить существование на источнике (запрос по car_id/sku_id)
      - Если не найдена или "продана" → пометить is_available = false
   b. Обновить БД батчем
   c. Пауза между батчами (1-2 минуты)
3. Завершение (повторить через 6-12 часов или по расписанию)
```

## ✅ Принятые решения

1. **Валидация и удаление**: Отдельный воркер в `datahub/`, помечать `is_available = false`
2. **Фильтрация по году**: ДО детального парсинга
3. **При совпадении ID**: Пропускать (не обновлять), оптимизировать чтобы не парсить старые
4. **Параллелизм**: 5 одновременных запросов для каждого источника

6. **Запуск**: По запросу (расписание отдельным сервисом)
7. **Частичные данные**: `power` обязателен - если нет power, пропускать машину
