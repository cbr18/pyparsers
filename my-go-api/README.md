# My Go API

This project is a simple API for managing cars, filters, and updates from external sources. It utilizes Swagger for API documentation and follows a clean architecture pattern.

## Project Structure

```
my-go-api
├── docs
│   └── swagger.json       # Swagger documentation for the API
├── cmd
│   └── main.go            # Entry point of the application
├── internal
│   ├── handlers
│   │   └── handlers.go    # HTTP handler functions for API endpoints
│   ├── models
│   │   └── models.go      # Data models used in the application
│   └── server
│       └── server.go      # HTTP server setup and middleware configuration
├── go.mod                  # Module definition and dependencies
└── README.md               # Project documentation
```

## Setup Instructions

1. **Clone the repository:**
   ```bash
   git clone <repository-url>
   cd my-go-api
   ```

2. **Install dependencies:**
   ```bash
   go mod tidy
   ```

3. **Run the application:**
   ```bash
   go run cmd/main.go
   ```

4. **Access the API documentation:**
   Open your browser and navigate to `http://localhost:8080/swagger/index.html` to view the API documentation generated by Swagger.

## Usage

- The API provides endpoints for managing cars and filters.
- Refer to the Swagger documentation for detailed information on available endpoints and their usage.
