# Автоматические миграции базы данных

## Что изменилось

Теперь при запуске datahub через Docker Compose автоматически применяются все необходимые миграции базы данных.

## Как это работает

1. **Отдельный контейнер миграций** (`datahub-migrate`) запускается первым
2. Ждет готовности PostgreSQL
3. Применяет все доступные миграции из папки `datahub/migrations/`
4. После успешного завершения миграций запускается основное приложение

## Использование

### Обычный запуск
```bash
docker-compose up -d
```

Миграции применятся автоматически перед запуском приложения.

### Просмотр логов миграций
```bash
docker-compose logs datahub-migrate
```

### Тестирование миграций
```bash
# Windows
.\test-migrations.ps1

# Linux/Mac
./test-migrations.sh
```

## Структура

- `datahub/migrate.Dockerfile` - Dockerfile для контейнера миграций
- `datahub/run-migrations.sh` - Скрипт применения миграций
- `datahub/wait-for-db.sh` - Скрипт ожидания готовности БД
- `datahub/migrations/` - Папка с файлами миграций

## Добавление новых миграций

1. Создайте новые файлы миграций в `datahub/migrations/`:
   ```
   000009_new_feature.up.sql
   000009_new_feature.down.sql
   ```

2. Перезапустите сервисы:
   ```bash
   docker-compose down
   docker-compose up -d
   ```

Новые миграции применятся автоматически при следующем запуске.

## Преимущества

- ✅ Автоматическое применение миграций
- ✅ Безопасный запуск (приложение не запустится при ошибке миграций)
- ✅ Логирование всех операций
- ✅ Поддержка отката миграций
- ✅ Совместимость с существующими миграциями
