# Рекомендации по паузам и ожиданию разблокировки

## 📊 Текущая ситуация:

### CHE168:
- **API блокировки**: 400+ ошибок 403 за 24 часа
- **Fallback работает**: 81 успешных раз (около 17% от блокировок)
- **Блокировки частые**, но fallback справляется

### DONGCHEDI:
- **Selenium блокировки**: 158 раз за 24 часа
- **Мобильный fallback**: 610 ошибок (не работает)
- **Блокировки частые**, но есть периоды когда работает

---

## 🤔 Стоит ли делать паузы?

### ✅ **ДА, СТОИТ, но с умом:**

#### 1. **CHE168 - API блокировки (403)**

**Рекомендация: КОРОТКИЕ паузы + использовать fallback**

- **Короткие паузы (5-15 секунд)** после 403 ошибки
- **Fallback работает** - используйте его как основной метод при блокировке
- **НЕ ждать долго** - блокировки могут длиться часами
- **Продолжать парсинг через fallback** вместо ожидания

**Почему:**
- Fallback успешно работает в 81 случае
- API может быть заблокирован долго (часы/дни)
- Fallback не требует API, поэтому можно продолжать работу

**Стратегия:**
- При 403 → сразу переходить на fallback
- Делать паузу 5-10 секунд перед следующим запросом
- НЕ ждать разблокировки API - использовать fallback как основной метод

---

#### 2. **DONGCHEDI - Selenium блокировки**

**Рекомендация: СРЕДНИЕ паузы + ротация методов**

- **Средние паузы (30-60 секунд)** после блокировки
- **Ротация методов**: Selenium → мобильный → HTML fallback
- **Мониторинг блокировок**: отслеживать частоту

**Почему:**
- Блокировки могут быть временными (минуты/часы)
- Мобильный fallback не работает (610 ошибок)
- HTML fallback работает частично

**Стратегия:**
- При блокировке → пауза 30-60 секунд
- Попробовать мобильный fallback
- Если не работает → пауза 5-10 минут
- После паузы → повторить попытку

---

## 🕐 Когда делать паузы?

### ✅ **Делать паузы:**

1. **После серии блокировок** (3-5 подряд):
   - Короткая пауза: 10-30 секунд
   - Средняя пауза: 1-5 минут

2. **При частых блокировках** (>10 за час):
   - Длинная пауза: 10-30 минут
   - Возможно временная блокировка

3. **Перед повторной попыткой API** (CHE168):
   - Короткая пауза: 5-10 секунд
   - Не спамить запросами

---

### ❌ **НЕ делать длинные паузы:**

1. **CHE168 с fallback**:
   - Fallback работает независимо от API
   - Можно продолжать парсинг через fallback
   - Не нужно ждать разблокировки API

2. **Когда есть рабочий fallback**:
   - Использовать fallback вместо ожидания
   - Продолжать работу
