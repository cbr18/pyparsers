# Резюме изменений: Система sort_number

## Что было добавлено

### 1. Новые поля в ответах API
- `sort_number` - порядковый номер машины (большие номера = новые машины)
- `source` - источник данных ('dongchedi' или 'che168')

### 2. Вспомогательная функция
```python
def _get_next_sort_number(existing_cars: List[Dict], source: str) -> int:
```
Определяет следующий номер для нумерации при инкрементальном обновлении.

### 3. Обновленные эндпоинты

#### Обычный парсинг
- `/cars/dongchedi` - добавляет `sort_number` (N, N-1, N-2...) и `source: 'dongchedi'`
- `/cars/che168` - добавляет `sort_number` (N, N-1, N-2...) и `source: 'che168'`
- `/cars/dongchedi/page/{page}` - аналогично
- `/cars/che168/page/{page}` - аналогично

#### Полный парсинг
- `/cars/dongchedi/all` - последовательная нумерация по убыванию через все страницы
- `/cars/che168/all` - последовательная нумерация по убыванию через все страницы

#### Инкрементальное обновление
- `/cars/dongchedi/incremental` - продолжает нумерацию от существующих машин
- `/cars/che168/incremental` - продолжает нумерацию от существующих машин

## Логика нумерации

1. **Обычный парсинг**: Если найдено N машин, то номера: N, N-1, N-2, ..., 1
2. **Полный парсинг**: Последовательная нумерация по убыванию (новые машины - большие номера)
3. **Инкрементальное обновление**: Продолжает от max(sort_number) + 1 для того же источника

**Результат**: При сортировке по убыванию `sort_number` получаем самые новые машины первыми.

## Использование на фронтенде

```javascript
// Сортировка по новизне
cars.sort((a, b) => b.sort_number - a.sort_number);

// Сортировка по источнику и новизне
cars.sort((a, b) => {
    if (a.source !== b.source) {
        return a.source.localeCompare(b.source);
    }
    return b.sort_number - a.sort_number;
});
```
