# Исправление логики инкрементального обновления

## ✅ Проблема

В методах `/incremental` нумерация шла по возрастанию, что приводило к неправильной сортировке по новизне:

```python
# СТАРАЯ логика (неправильная)
for car in new_cars:
    car['sort_number'] = next_sort_number  # 6, 7, 8, 9...
    next_sort_number += 1
```

**Результат**: При сортировке по убыванию самые новые машины оказывались в конце списка.

## 🔧 Решение

Изменили логику так, чтобы среди новых машин самые новые получали большие номера:

```python
# НОВАЯ логика (правильная)
total_new_cars = len(new_cars)
for i, car in enumerate(new_cars):
    car['sort_number'] = next_sort_number + total_new_cars - i - 1
```

## 📊 Пример работы

### Исходные данные:
- **Существующие машины**: max(sort_number) = 5
- **Новые машины** (отсортированные по новизне):
  1. Новая машина A (самая новая)
  2. Новая машина B
  3. Новая машина C (самая старая из новых)

### СТАРАЯ логика:
```
next_sort_number = 6
Новая машина A → sort_number: 6
Новая машина B → sort_number: 7  
Новая машина C → sort_number: 8
```

### НОВАЯ логика:
```
next_sort_number = 6
total_new_cars = 3
Новая машина A → sort_number: 6 + 3 - 0 - 1 = 8
Новая машина B → sort_number: 6 + 3 - 1 - 1 = 7
Новая машина C → sort_number: 6 + 3 - 2 - 1 = 6
```

### Результат сортировки по убыванию:

**СТАРАЯ логика** (неправильно):
1. Новая машина C (sort_number: 8) ← самая старая из новых
2. Новая машина B (sort_number: 7)
3. Новая машина A (sort_number: 6) ← самая новая из новых
4. Существующая (sort_number: 5)

**НОВАЯ логика** (правильно):
1. Новая машина A (sort_number: 8) ← самая новая из новых
2. Новая машина B (sort_number: 7)
3. Новая машина C (sort_number: 6) ← самая старая из новых
4. Существующая (sort_number: 5)

## 🚀 Формула

```python
sort_number = next_sort_number + total_new_cars - i - 1
```

Где:
- `next_sort_number` = max(существующие sort_number) + 1
- `total_new_cars` = количество новых машин
- `i` = индекс машины в списке новых машин (0, 1, 2, ...)

## ✅ Преимущества исправления

1. **Правильная сортировка**: При сортировке по убыванию получаем самые новые машины первыми
2. **Консистентность**: Логика одинакова для всех типов парсинга
3. **Интуитивность**: Большие номера = новые машины
4. **Корректность инкрементального обновления**: Новые машины получают номера больше существующих

## 🎯 Итоговый результат

Теперь все методы API работают с единой логикой:
- **Новые машины** = большие номера
- **Старые машины** = маленькие номера
- **Сортировка по убыванию** = самые новые первыми
- **Инкрементальное обновление** = корректное продолжение нумерации с правильным порядком
