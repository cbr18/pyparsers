# Task API Documentation

## Обзор

Сервис datahub теперь поддерживает фоновое выполнение задач обновления источников данных. Все операции обновления выполняются асинхронно в фоне, что позволяет не блокировать API запросы.

## Новые эндпоинты

### 1. Создание задач

#### `GET /update/{source}/full`
Создает задачу полного обновления для указанного источника.

**Параметры:**
- `source` (path) - источник данных: `dongchedi` или `che168`

**Ответ:**
```json
{
  "task_id": "uuid-string"
}
```

#### `POST /update/{source}`
Создает задачу инкрементального обновления для указанного источника.

**Параметры:**
- `source` (path) - источник данных: `dongchedi` или `che168`

**Ответ:**
```json
{
  "task_id": "uuid-string"
}
```

### 2. Управление задачами

#### `GET /tasks/{id}`
Получает информацию о конкретной задаче.

**Параметры:**
- `id` (path) - ID задачи (UUID)

**Ответ:**
```json
{
  "id": "uuid-string",
  "source": "dongchedi",
  "type": "full",
  "status": "done",
  "result": {
    "count": 150,
    "message": "Successfully updated 150 cars"
  },
  "error": "",
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:35:00Z"
}
```

#### `GET /tasks`
Получает список всех задач.

**Ответ:**
```json
{
  "data": [
    {
      "id": "uuid-string-1",
      "source": "dongchedi",
      "type": "full",
      "status": "done",
      "result": {...},
      "created_at": "2024-01-15T10:30:00Z",
      "updated_at": "2024-01-15T10:35:00Z"
    },
    {
      "id": "uuid-string-2",
      "source": "che168",
      "type": "incremental",
      "status": "in_progress",
      "created_at": "2024-01-15T10:40:00Z",
      "updated_at": "2024-01-15T10:40:00Z"
    }
  ]
}
```

## Статусы задач

- `pending` - задача создана, ожидает выполнения
- `in_progress` - задача выполняется
- `done` - задача успешно завершена
- `failed` - задача завершилась с ошибкой

## Типы задач

- `full` - полное обновление (очистка старых данных + загрузка новых)
- `incremental` - инкрементальное обновление (добавление только новых данных)

## Источники данных

- `dongchedi` - источник Dongchedi
- `che168` - источник Che168

## Примеры использования

### 1. Запуск полного обновления
```bash
curl -X GET "http://localhost:8080/update/dongchedi/full"
# Ответ: {"task_id": "123e4567-e89b-12d3-a456-426614174000"}
```

### 2. Запуск инкрементального обновления
```bash
curl -X POST "http://localhost:8080/update/che168"
# Ответ: {"task_id": "123e4567-e89b-12d3-a456-426614174001"}
```

### 3. Проверка статуса задачи
```bash
curl -X GET "http://localhost:8080/tasks/123e4567-e89b-12d3-a456-426614174000"
```

### 4. Получение списка всех задач
```bash
curl -X GET "http://localhost:8080/tasks"
```

## Особенности реализации

1. **Фоновое выполнение**: Все задачи выполняются в отдельных goroutines
2. **Хранение в памяти**: Задачи хранятся в памяти с использованием sync.Mutex для thread-safety
3. **Сохранение существующей логики**: Вся существующая логика работы с источниками и БД остается неизменной
4. **Автоматическое обновление статуса**: Статус задач обновляется автоматически по мере выполнения
5. **Обработка ошибок**: Ошибки сохраняются в поле `error` задачи

## Мониторинг

Для мониторинга состояния задач используйте эндпоинт `GET /tasks` для получения списка всех задач и их текущих статусов.
